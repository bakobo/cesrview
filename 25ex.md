PRINT MODEL — SETTLED (Model A): print filters the mounted page. JS sets data-print-scope on
<html>; pure @media print CSS keys off it and hides the other components. One printout = exactly
ONE top-level component: {transcript | manifest | one decoded event}. Exhibit is ONE event at a
time (3 events = 3 printouts) — NOT expand-all. Transcript and manifest are whole-stream, single
shot. No off-screen rendering; no dedicated print views (Model B rejected). No multi-select /
report-builder (rule of three).

EXHIBIT REPRESENTATION — SETTLED + PROBE-CONFIRMED: identity values stay rendered as entviz pills
on paper (NOT expanded to full text). Probe /tmp/print-probe.mjs (grayscale-on-white of
samples/multisig-oobi.cesr) showed pills survive monochrome: the pill carries a first4..mid4..last4
truncated stub (CONSTANT footprint even for 88-char sigs and thousands-char post-quantum
primitives) plus avatar geometry (circle count/position + luminosity bands) that survive grayscale
AND a harsh-photocopier contrast pass; identical values render identical pills (matchable). Full
values live ONCE in an optional raw-CESR appendix per exhibit, not per field row. Killed the
earlier "drop pills, show full identifiers" idea (it repeats the transcript and breaks on long
primitives).

DERIVED PRINT PRINCIPLES: (1) width-stability — represent identities as bounded-footprint tokens,
never inline variable-length text. (2) Print always renders the light/paper palette regardless of
the active data-theme. (3) Hide interactive-only affordances in print (the pill locate "#" glyph;
onLocate is meaningless on a one-event page) but keep informational glyphs (typeSignal scheme
mark). (4) Recognition on paper rides truncated-text + geometry, never hue. (5) Each printout
self-identifies (exhibit: event SAID + controller AID + sn) and carries a posture band
"structure decoded, not verified" (b6zx2d, @v7kd3m). (6) Unbreakable structural units
(break-inside: avoid) — subject to the open pagination risk below.

TRIGGER — PROVISIONAL (two-way door): native Ctrl+P / File>Print pegged to the exhibit; the other
two scopes exposed via the ~3xh2 right-click menu. Print RIDES the ~3xh2 context-menu decision, it
does not drive it; hover-glyph-in-chrome fallback if ~3xh2 does not add custom context menus.

SHARED LAYER: print and mobile share ONLY a narrow single-column layer (refactor into ONE shared
thing so they cannot drift). They DIVERGE on interaction (mobile keeps it, print strips it) and
color (mobile keeps, print monochrome). Mobile layout is currently untested.

OPEN BEFORE IMPLEMENT: pagination / page-break behavior, incl. an event taller than a page and
long-primitive wrapping in the transcript (needs a real page.pdf() probe, not screenshots);
confirm transcript + manifest print clean; check entviz's own monochrome/print prop; decide the
shared-layer CSS architecture; enumerate per-scope empty/error states (fail-closed).

ORACLE: chromium page.pdf() + emulateMedia('print') via the existing Playwright recipe;
grayscale-CSS approximation used so far (/tmp/print-probe.mjs).
